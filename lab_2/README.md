# Лабораторная работа №2

## Тема
Разработка оконного приложения на `Python` с графическим пользовательским интерфейсом с использованием шаблона `Model-View-Controller`.

## Цель работы
- Изучить принципы построения графического пользовательского интерфейса.
- Разработать программную систему с графическим пользовательским интерфейсом на языке `Python`.

## Назначение программы
Приложение предназначено для хранения, просмотра, поиска, удаления, импорта и экспорта записей о спортсменах.  
В качестве хранилища используется база данных `SQLite`, а для обмена данными с файлами используется формат `XML`.

## Используемые технологии
- `Python`
- `PyQt5`
- `SQLite`
- `XML DOM` для записи XML
- `XML SAX` для чтения XML

## Архитектура проекта
Проект построен по шаблону `Model-View-Controller`.

### Model
- `models/sportsman.py`  
  Описывает сущность спортсмена.
- `models/xml_handler.py`  
  Отвечает за сохранение данных в XML через DOM и чтение из XML через SAX.
- `database/database.py`  
  Отвечает за работу с базой данных SQLite.

### View
- `views/main_window.py`  
  Главное окно приложения.
- `views/add_dialog.py`  
  Диалог добавления записи.
- `views/search_dialog.py`  
  Диалог поиска записей.
- `views/delete_dialog.py`  
  Диалог удаления записей.
- `views/pagination_widget.py`  
  Виджет постраничного вывода.

### Controller
- `controllers/controller.py`  
  Связывает интерфейс, базу данных и обработку XML.

## Формат записи
Каждая запись о спортсмене содержит поля:
- ФИО
- Команда
- Позиция
- Количество титулов
- Вид спорта
- Разряд

Поле `Количество титулов` хранится как целое число и вводится через `QSpinBox`.

## Возможности программы
- Добавление записи через отдельное диалоговое окно.
- Поиск записей через отдельное диалоговое окно.
- Удаление записей через отдельное диалоговое окно.
- Вывод всех записей в главном окне.
- Альтернативное отображение записей в виде дерева.
- Хранение записей в базе данных `SQLite`.
- Сохранение записей в `XML`.
- Загрузка записей из `XML`.
- Поддержка двух режимов импорта XML.
- Постраничный вывод записей в главном окне и в окне поиска.
- Вызов основных команд через меню и панель инструментов.

## Диалог добавления записи
Диалог добавления позволяет ввести данные о спортсмене вручную.

Особенности:
- ФИО и вид спорта проверяются на непустое значение.
- Количество титулов вводится через `QSpinBox`.
- Разряд выбирается из списка, но допускает ручной ввод.

## Диалог поиска
Поиск выполняется в отдельном окне, а результаты выводятся в нём же.

Поддерживаются три режима поиска:
- по ФИО или виду спорта;
- по диапазону количества титулов;
- по ФИО или разряду.

Результаты поиска выводятся в таблице с постраничным выводом.

## Диалог удаления
Удаление выполняется в отдельном диалоговом окне.

Поддерживаются три режима удаления:
- по ФИО или виду спорта;
- по диапазону количества титулов;
- по ФИО или разряду.

После удаления пользователю выводится сообщение:
- сколько записей удалено;
- либо сообщение о том, что записи по указанным условиям не найдены.

## Постраничный вывод
Постраничный вывод реализован:
- в главном окне;
- в окне поиска.

Виджет пагинации поддерживает:
- переход на первую страницу;
- переход на предыдущую страницу;
- переход на следующую страницу;
- переход на последнюю страницу;
- выбор количества записей на странице;
- отображение текущей страницы;
- отображение общего количества страниц;
- отображение общего количества записей;
- отображение числа записей, показываемых на странице.

## Работа с базой данных
Все записи хранятся в базе данных `SQLite`.

База данных содержит таблицу `sportsmen` со следующими полями:
- `id`
- `full_name`
- `team`
- `position`
- `titles`
- `sport`
- `rank`

## Работа с XML
Для хранения данных в файле используется формат `XML`.

### Запись XML
Сохранение выполняется через `DOM`:
- создаётся документ;
- создаётся корневой узел `sportsmen`;
- для каждой записи создаётся узел `sportsman`;
- внутри него создаются поля записи.

### Чтение XML
Чтение выполняется через `SAX`:
- используется обработчик `SportsmanHandler`;
- данные читаются последовательно из XML-файла;
- по завершении чтения формируется список объектов `Sportsman`.

## Режимы импорта XML
При загрузке XML пользователь выбирает один из двух режимов:

### 1. Добавить из XML
- существующие записи в базе данных сохраняются;
- из XML добавляются только новые записи;
- дубликаты пропускаются.

### 2. Заменить данными из XML
- база данных очищается;
- затем загружаются записи из XML-файла.

### Проверка дубликатов
При импорте в режиме добавления дубликаты определяются по сочетанию:
- фамилия;
- имя.

Если запись с таким сочетанием уже есть в базе данных или встречается повторно в текущем XML-файле, она не добавляется.

## Альтернативное отображение данных
В главном окне записи могут отображаться:
- в виде таблицы;
- в виде дерева.

В древовидном представлении:
- верхний узел соответствует конкретному спортсмену;
- дочерние элементы соответствуют полям записи;
- листовой элемент дерева соответствует отдельному полю записи, что соответствует условию лабораторной работы.
